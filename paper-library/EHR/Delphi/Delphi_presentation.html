1.1 多病共存建模的重要性

1.1 Why Multimorbidity Modeling Matters

人类疾病的进展具有显著的时间依赖性和共病模式：健康期、急性发病、慢性衰退在个体间分布不均，多病共存与生活方式、遗传特征、社会经济状况高度相关。理解个体化的多病共存风险对医疗决策、生活方式干预、筛查入组至关重要。

Human disease progression exhibits strong temporal dependencies and comorbidity patterns: healthy periods, acute onset, and chronic decline are unevenly distributed across individuals. Multimorbidity correlates strongly with lifestyle, genetic traits, and socioeconomic status. Understanding individual multimorbidity risk is critical for clinical decisions, lifestyle interventions, and screening enrollment.

2.2 Delphi 架构：对 GPT-2 的三项关键改造

2.2 Delphi Architecture: Three Key GPT-2 Modifications

基于 nanoGPT (Karpathy) 实现的 GPT-2，针对健康轨迹建模做了三项核心改造：

Based on nanoGPT (Karpathy) implementing GPT-2, with three core modifications for health trajectory modeling:

Transformer Blocks Transformer Blocks 改造 3: 修改的因果掩码 Mod 3: Modified causal mask

改造 1: 连续时间的年龄编码

Modification 1: Continuous-Time Age Encoding

GPT 的离散位置编码被替换为连续年龄编码，使用不同频率的 sin/cos 基函数编码年龄（以天为单位）：

GPT's discrete positional encoding is replaced by continuous age encoding, using sin/cos basis functions of different frequencies to encode age in days:

$$\text{AgeEnc}(t) = W \cdot [\sin(\omega_1 t),\, \cos(\omega_1 t),\, \sin(\omega_2 t),\, \cos(\omega_2 t),\, \ldots]$$
  • 最低频率：$\omega_{\min} = 1/365$（年周期）
  • Minimum frequency: $\omega_{\min} = 1/365$ (yearly period)
  • $W$ 是可训练的线性变换，允许模型跨维度共享基函数
  • $W$ is a trainable linear projection, allowing the model to share basis functions across dimensions
  • 优势：可处理任意长度的输入序列，不受位置编码固定长度限制
  • Advantage: Handles arbitrary-length sequences without the fixed-length constraint of positional encoding

改造 2: 指数等待时间模型（双输出头）

Modification 2: Exponential Waiting Time Model (Dual Output Heads)

标准 GPT 只预测"下一个 token 是什么"（softmax）。Delphi 增加了一个输出头，同时预测"距下一个事件多久"。

Standard GPT only predicts "what is the next token" (softmax). Delphi adds a second output head that simultaneously predicts "how long until the next event".

数学基础：竞争指数模型 (Competing Exponentials)

Mathematical Foundation: Competing Exponentials

设 $T_i$ 是到第 $i$ 个可能事件的等待时间，$\lambda_i = \exp(\text{logit}_i)$ 为发病率。若 $T_i \sim \text{Exp}(\lambda_i)$，则：

Let $T_i$ be the waiting time to event $i$, with incidence rate $\lambda_i = \exp(\text{logit}_i)$. If $T_i \sim \text{Exp}(\lambda_i)$, then:

Token 分类（下一个事件是 $j$ 的概率）：

Token classification (probability that the next event is $j$):

$$P\!\left(j = \arg\min_i T_i\right) = \frac{\lambda_j}{\sum_i \lambda_i} = \text{softmax}(\text{logits})_j$$

时间预测（到下一事件的时间 $T^* = \min_j T_j$ 服从参数 $\lambda^* = \sum_i \lambda_i$ 的指数分布）：

Time prediction (time to next event $T^* = \min_j T_j$ follows exponential with rate $\lambda^* = \sum_i \lambda_i$):

$$\mathcal{L}_T = -\log p(T^*) = -\!\left(\log\!\sum_i e^{\text{logit}_i} - \sum_i e^{\text{logit}_i} \cdot T^*\right)$$

总损失：$\mathcal{L} = \mathcal{L}_j + \mathcal{L}_T$（token 分类损失 + 时间预测损失）

Total loss: $\mathcal{L} = \mathcal{L}_j + \mathcal{L}_T$ (token classification + time prediction)

关键洞察：$\lambda_i = \exp(\text{logit}_i)$ 可直接解释为每日发病率！这使模型输出具有明确的流行病学含义——不仅预测"下一个是什么"，还给出每种疾病的绝对发病率。 Key Insight: $\lambda_i = \exp(\text{logit}_i)$ is directly interpretable as the daily incidence rate! This gives model outputs clear epidemiological meaning — predicting not only "what comes next" but also the absolute incidence rate of each disease.

改造 3: 修改的因果注意力掩码

Modification 3: Modified Causal Attention Mask

标准 GPT 只能关注序列中"前面"的 token。Delphi 进一步：

Standard GPT attends only to earlier tokens. Delphi additionally:

  • 额外掩蔽同一时间记录的 token（同时诊断的疾病不应互相看到）
  • Masks tokens recorded at the same time (simultaneously diagnosed diseases should not attend to each other)
  • 非信息性 padding token 不参与对其他 token 的预测
  • Non-informative padding tokens are excluded from predicting other tokens

权重绑定 (Weight Tying)

Weight Tying

Token embedding 矩阵和最终线性层共享权重，减少参数量，并使输入/输出 embedding 可一致解释（用于 UMAP 可视化和可解释性分析）。

Token embedding matrix and final linear layer share weights, reducing parameters and enabling consistent interpretation of input/output embeddings (used for UMAP visualization and explainability).

2.5 推理与采样

2.5 Inference & Sampling

预测模式

Prediction Mode

给定部分健康轨迹作为 prompt，计算每种 token 的后续发病率（每日），提供个体化风险评估

Given a partial health trajectory as prompt, compute per-token incidence rates (daily), providing individualized risk assessment

采样模式（生成轨迹）

Sampling Mode (Trajectory Generation)

  1. 根据 softmax 分布采样下一个 token $j$
  2. Sample next token $j$ from softmax distribution
  3. 从 $\text{Exp}(\lambda_j)$ 采样等待时间
  4. Sample waiting time from $\text{Exp}(\lambda_j)$
  5. 迭代进行，生成完整的未来健康轨迹
  6. Iterate to generate a complete future health trajectory

4.3 合成数据训练

4.3 Synthetic Data Training

重要发现：在完全合成数据上训练的 Delphi，验证 AUC 为 0.74（原始 0.77），仅损失 3 个百分点。

Key finding: Delphi trained on fully synthetic data achieves validation AUC of 0.74 (vs. 0.77 original), losing only 3 percentage points.

合成数据保留了训练 Delphi 所需的大部分信息，可作为隐私保护的替代数据源，具有重要的数据共享意义。

Synthetic data retains most information needed to train Delphi, making it a viable privacy-preserving alternative data source with significant implications for data sharing.

2
数据源缺失偏差 Data Source Missingness Bias

不同数据源对不同参与者覆盖不一致。住院记录特有的疾病（如脓毒症 93% 来自住院），在有住院历史的个体中预测率高 8-10 倍——这是数据收集过程的伪影，而非真实共病关联。

Coverage varies across participants by data source. Hospital-specific diseases (e.g., sepsis 93% from hospital admissions) are predicted 8-10x higher for individuals with prior hospitalization — this is a data collection artifact, not a true comorbidity association.

3
年龄和随访限制 Age and Follow-up Limitations

80 岁以上数据有限，该年龄段不可靠建模；随访时间不均匀影响长期预测的可靠性。

Limited data above age 80, making modeling unreliable in that age range; uneven follow-up times affect long-term prediction reliability.

因果推断警告：Delphi 建模的是时间依赖关系而非因果关系，不应用于指导干预研究或推断治疗效果。 Causal Inference Warning: Delphi models temporal dependencies, not causal relationships, and should not be used to guide intervention studies or infer treatment effects.
6

讨论与思考

Discussion & Reflections

优雅的数学框架：竞争指数模型的自洽性 Elegant Mathematical Framework: Self-consistency of Competing Exponentials

竞争指数模型天然对应 softmax 分类 + 指数等待时间，数学上完美自洽：softmax 给出事件类型的概率，指数分布给出等待时间，两者共享同一组 logits（发病率参数）。这不是一个工程技巧，而是统计物理中竞争过程的严格数学结论。

The competing exponentials model naturally corresponds to softmax classification + exponential waiting time, mathematically self-consistent: softmax gives event type probability, exponential distribution gives waiting time, and both share the same logits (incidence rate parameters). This is not an engineering trick but a rigorous mathematical result from competing processes in statistical physics.

2M 参数是否太小？Scaling 的天花板 Is 2M Parameters Too Small? The Scaling Ceiling

模型容量受限于 UK Biobank 的 50 万人规模——这是数据瓶颈，而非模型瓶颈。如果数据量扩大（如全英国 NHS 数据，>6000 万人），能否训练更大、更强的模型？当前 1,268 token 的词汇量是否过于粗糙？ICD-10 Level 4/5 会如何？这些都是 Delphi 范式能否突破现有上限的关键问题。

Model capacity is constrained by UK Biobank's 500K-person scale — a data bottleneck, not a model bottleneck. If data expands (e.g., full UK NHS data, >60M people), could larger, more powerful models be trained? Is the current 1,268-token vocabulary too coarse? What about ICD-10 Level 4/5? These are key questions for whether the Delphi paradigm can break its current ceiling.

生活方式 token 时间随机化的影响 Impact of Lifestyle Token Time Randomization

将 BMI/吸烟/饮酒的时间点在 ±20~+40 年内随机化是合理的工程选择（避免与入组时的 immortal time bias 混淆）。但这可能削弱了生活方式变化的真实时间效应（如戒烟后风险下降的时间模式）。这类 "入组偏差的工程绕行" 是 EHR 模型的普遍挑战。

Randomizing BMI/smoking/alcohol timestamps within ±20~+40 years is a reasonable engineering choice (avoiding immortal time bias at enrollment). But this may weaken the true temporal effects of lifestyle changes (e.g., time pattern of risk reduction after smoking cessation). This "engineering bypass of enrollment bias" is a common challenge in EHR modeling.

数据源偏差是根本性问题 Data Source Bias Is a Fundamental Problem

有住院历史的人被预测更高的疾病率，很多是数据可见性而非真实共病。这个问题在所有 EHR-based 模型中都存在，Delphi 的贡献是显式暴露了它（Figure 5 专门分析）。但如何去偏？论文未提供解决方案。一个方向：通过数据源 token 化，让模型学习"哪些预测是由数据可见性驱动的"。

Individuals with prior hospitalization are predicted higher disease rates — often due to data visibility, not true comorbidity. This problem exists in all EHR-based models; Delphi's contribution is explicitly exposing it (dedicated Figure 5 analysis). But how to debias? The paper provides no solution. One direction: tokenize data sources to let the model learn "which predictions are driven by data visibility".

临床部署的可行性 Clinical Deployment Feasibility

预测 >1,000 种疾病的发病率，如何呈现给临床医生？信息过载风险极高。论文缺乏个体预测的不确定性量化（置信区间只在 AUC 层面提供），个体预测的 calibration 如何（论文只展示群体水平）。此外，模型在 Delphi 的"可解释性"主要是事后分析（SHAP），临床医生更需要前向可解释性。

Predicting >1,000 disease incidence rates — how to present to clinicians? Information overload risk is high. The paper lacks individual-level uncertainty quantification (confidence intervals only at AUC level), and individual calibration is undemonstrated (only population-level shown). Furthermore, Delphi's "explainability" is primarily post-hoc analysis (SHAP); clinicians need forward explainability.

与病理 FM 的潜在交集（RAE 视角） Potential Intersection with Pathology FMs (RAE Perspective)

Delphi 从 EHR 预测疾病轨迹；病理基础模型（如 LPFM、PixCell）从组织图像辅助诊断。两者是否可以互补？设想：Delphi 识别高风险个体 → 病理 FM 辅助精确诊断 → 结果反馈更新 Delphi 的健康轨迹。多模态整合（Delphi EHR embedding + 病理图像特征）是否能进一步提升预测精度？

Delphi predicts disease trajectories from EHR; pathology foundation models (LPFM, PixCell) assist diagnosis from tissue images. Can they complement each other? Imagine: Delphi identifies high-risk individuals → pathology FM aids precise diagnosis → outcomes feed back to update Delphi's trajectories. Could multimodal integration (Delphi EHR embeddings + pathology image features) further improve prediction accuracy?
