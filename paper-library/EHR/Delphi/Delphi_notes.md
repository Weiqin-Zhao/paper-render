## 1. Motivation & Problem

### 1.1 多病共存建模的重要性

人类疾病的进展具有显著的**时间依赖性和共病模式**：
- 健康期、急性发病、慢性衰退在个体间分布不均
- 多病共存 (multimorbidity) 与生活方式、遗传特征、社会经济状况相关
- 理解个体化的多病共存风险对医疗决策、生活方式干预、筛查入组至关重要
- ICD-10 编码系统包含 **>1,000 种**顶级诊断——当前极少有模型能同时预测全谱疾病

### 1.2 人口老龄化的紧迫性

- 预计 2050 年全球癌症诊断数增加 77%
- 英国到 2040 年工作年龄慢性病患者从 300 万增至 370 万
- 建模疾病负担对医疗规划和经济决策至关重要

### 1.3 现有方法的局限

- **碎片化**：每种疾病各自独立的风险模型（如 QRISK3 仅用于心血管病）
- **时间建模困难**：传统流行病学模型难以处理事件间的时间依赖性
- **缺乏生成能力**：无法采样未来健康轨迹
- BERT-based 模型（Med-BERT, BEHRT, Hi-BEHRT）用于特定预测任务，但未充分评估**全面的、生成式的**多病共存建模

### 1.4 本文定位

将 GPT 架构改造为健康轨迹建模工具 **Delphi-2M**，实现：
- **同时预测 >1,000 种疾病**的发病率，基于个体过往病史
- **生成未来健康轨迹**，提供长达 20 年的疾病负担估计
- **可解释性分析**，揭示疾病间的共病依赖和时间效应
- **跨国验证**，在丹麦 190 万人口数据上零调参验证

核心 insight: 疾病序列与自然语言在结构上类似——都是基于上下文预测下一个 token。但关键区别是疾病发生在**连续时间轴**上，需要对 GPT 架构进行特定改造。

## 2. Method

### 2.1 数据表示：健康轨迹的 Token 化

将个体健康轨迹表示为 **(token, age)** 对的序列：

| Token 类别 | 数量 | 说明 |
|-----------|------|------|
| 疾病 token | 1,256 | ICD-10 Level 3 诊断码 + 死亡 |
| 性别 token | 2 | Male / Female，出生时记录 |
| 生活方式 token | 9 | BMI (3 档)、吸烟 (3 档)、饮酒 (3 档) |
| No-event padding | 1 | 以平均每 5 年 1 次的频率随机插入 |
| **总词汇量** | **1,258** | |

> **[Figure 1a: 健康轨迹示意图]**
> Caption: 基于 ICD-10 诊断、生活方式和 padding token 的健康轨迹示意，每个 token 在特定年龄记录。

**Padding token 的设计动机**：
- 年轻时期疾病事件稀少，长时间间隔导致模型无法更新基线风险
- 插入 padding token 消除长间隔，让模型能在任意时间点提供预测
- 消融实验 (Extended Data Fig. 1) 证实 padding 显著降低 loss，尤其在年轻年龄段

**生活方式 token 的时间处理**：
- BMI、吸烟、饮酒在 UK Biobank 入组时记录
- 为避免与入组时的 immortal time bias 混淆，将这些 token 的时间点在入组时间 ±20 至 +40 年内随机化
- 这些 token 仅作为输入，模型不预测它们

### 2.2 Delphi 架构：对 GPT-2 的三项关键改造

> **[Figure 1c: Delphi 模型架构图]**
> Caption: Delphi 架构。红色元素标注了相对于 GPT-2 的改动。

基于 nanoGPT (Karpathy) 实现的 GPT-2，做了三项改造：

#### 改造 1: 连续时间的年龄编码（替代位置编码）

GPT 的**位置编码**标识每个 token 的离散位置。Delphi 替换为**年龄编码**：

$$\text{AgeEnc}(t) = W \cdot [\sin(\omega_1 t), \cos(\omega_1 t), \sin(\omega_2 t), \cos(\omega_2 t), \ldots]$$

- 使用不同频率的 sin/cos 基函数编码年龄（以天为单位）
- 最低频率 $1/365$
- $W$ 是可训练的线性变换，允许模型跨维度共享基函数
- **优势**：可处理任意长度的输入（不再受位置编码的固定长度限制）

#### 改造 2: 指数等待时间模型（多一个输出头）

标准 GPT 只用 softmax 预测"下一个 token 是什么"。Delphi 增加了一个输出头预测"距下一个 token 多久"。

**数学基础 —— 竞争指数模型 (Competing Exponentials)**：

设 $T_i$ 是从当前事件到第 $i$ 个可能事件的等待时间，$\lambda_i = \exp(\text{logit}_i)$。若 $T_i \sim \text{Exp}(\lambda_i)$，则：

**Token 分类**：下一个事件是 $j$ 的概率为：

$$P(j = \arg\min T_i) = \frac{\lambda_j}{\sum_i \lambda_i} = \text{softmax}(\text{logits})_j$$

这恰好就是标准的 cross-entropy 分类！

**时间预测**：到下一事件的时间 $T^* = \min T_j$ 服从指数分布，参数 $\lambda^* = \sum_i \lambda_i$：

$$\mathcal{L}_T = -\log p(T^*) = -(\log\sum_i e^{\text{logit}_i} - \sum_i e^{\text{logit}_i} \times T^*)$$

**总损失**：$\mathcal{L} = \mathcal{L}_j + \mathcal{L}_T$

> **关键 insight**：$\lambda_i = \exp(\text{logit}_i)$ 可直接解释为每日发病率！这使模型输出具有明确的流行病学含义——不仅预测"下一个是什么"，还给出每种疾病的绝对发病率。

#### 改造 3: 因果注意力掩码的修改

标准 GPT: 只能关注序列中"前面"的 token。Delphi 修改为：
- 额外掩蔽**同一时间记录**的 token（同时诊断的疾病不应互相看到）
- 非信息性 padding token 不参与对其他 token 的预测

#### 权重绑定 (Weight Tying)

Token embedding 矩阵和最终线性层共享权重。好处：
- 减少参数量
- 输入/输出 embedding 可以一致地解释（用于 UMAP 可视化和可解释性分析）

### 2.3 模型缩放与超参数

> **[Figure 1e: 缩放定律]**
> Caption: Delphi 的缩放定律——最优验证 loss 随模型参数和训练数据量的变化。

通过系统的超参数搜索（Extended Data Fig. 2）确定：
- **最优模型大小**：约 **200 万参数**（对应 UK Biobank 的数据规模）
- **选定配置**：embedding dim = 120, 12 layers, 12 heads → **2.2M 参数**
- 验证了 LLM 领域的 scaling laws 在健康数据上同样成立

### 2.4 训练配置

| 项目 | 详情 |
|------|------|
| 优化器 | Adam, 标准参数 |
| 迭代次数 | 200,000 |
| Batch size | 128 |
| 学习率 | 6e-4 → 6e-5 (cosine decay, 1000 step warmup) |
| 精度 | 32-bit float |
| 训练数据 | 402,799 人 (UK Biobank 80%) |

### 2.5 消融分析

> **[Figure 1f: 消融结果]**
> Caption: 各项改造相对于年龄-性别基线的 cross-entropy 改善。

| 模型变体 | 贡献 |
|---------|------|
| NanoGPT-2M (原始) | 基线 |
| + Padding tokens | 大幅改善，尤其年轻年龄段 |
| + 年龄编码 + 掩码修改 | 进一步改善 |
| + 指数时间模型 (Delphi-2M) | 关键区分——能计算绝对发病率，提供一致的时间间隔估计 |

### 2.6 推理与采样

**预测模式**：给定部分健康轨迹（prompt），计算每种 token 的后续发病率（每日）。

**采样模式**（生成轨迹）：
1. 根据 softmax 分布采样下一个 token $j$
2. 从 $\text{Exp}(\lambda_j)$ 采样等待时间
3. 迭代进行，生成完整的未来健康轨迹

## 3. 数据

### 3.1 UK Biobank

- **总规模**：502,485 人，7,426,036 tokens
- **训练集**：402,799 人 (80%)，数据截至 2020.07.01
- **验证集**：100,639 人 (20%)，同期
- **纵向测试集**：471,057 人（2020.07.01 仍存活），评估 2021.07–2022.07 的发病
- **入组年龄**：37-73 岁（2006-2010 入组）

数据来源（UK Biobank Category 1712）：
- 自我报告 (self-reported)
- 初级保健记录 (primary care / GP)
- 住院记录 (hospital admissions)
- 癌症登记 (cancer registry)
- 死亡登记 (death registry)

### 3.2 丹麦疾病登记

- **规模**：1,931,630 人（50-80 岁，截至 2016.01.01），11,512,665 tokens
- **时间跨度**：1978-2018
- **用途**：纯外部验证，模型**零调参**直接迁移
- **缺失**：无生活方式数据（BMI、吸烟、饮酒 token 视为缺失）
- ICD-8 到 ICD-10 的映射使用已发表的单向映射

### 3.3 数据划分策略

| 数据集 | 用途 | 评估方式 |
|-------|------|---------|
| UKB 训练 (80%) | 模型训练 | - |
| UKB 验证 (20%) | 超参数优化、模型评估 | 横截面 AUC |
| UKB 纵向测试 (94%) | 前瞻性评估 | 设 1 年 gap，评估下一年 |
| 丹麦 (190 万) | 外部验证 | 同上，零调参 |

## 4. 实验结果

### 4.1 多疾病发病率预测

> **[Figure 2: 多疾病发病率预测结果]**
> Caption: Delphi-2M 对多种疾病发病率的预测。(a) 9 种疾病和死亡的预测率 vs 年龄；(b-d) AUC 按疾病、ICD-10 章节、性别汇总；(e) 不同预测时间窗的 AUC；(f) 与临床风险评分的 ROC 对比。

**核心结果**：

- **平均 AUC ≈ 0.76**（年龄-性别分层，验证集）
- **97% 的诊断 AUC > 0.5**，说明绝大多数疾病具有至少部分可预测性
- **死亡预测 AUC = 0.97**（两性）
- AUC 随预测时间窗增加而下降：0.76 (next event) → 0.70 (10 年后)，但仍远超基线
- **校准良好**：预测发病率与观测数量在各 5 年年龄段中高度一致

**与现有临床模型对比** (Figure 2f)：

| 疾病 | Delphi AUC | 最佳对比方法 | 对比 AUC |
|------|-----------|------------|---------|
| 心血管病 (CVD) | 0.70 | QRisk3 | 0.71 |
| 痴呆 | 0.82 | UKBDRS | 0.81 |
| 死亡 | 0.81 | Charlson | 0.73 |
| 糖尿病 | 较差 | HbA1c (单指标) | 更好 |

- Delphi 在**大多数疾病**上匹配或超越单病种模型
- CVD 上略逊于 QRisk3（0.70 vs 0.71），因为 QRisk3 使用更丰富的临床变量
- **糖尿病**上 Delphi 不如 HbA1c——标志着未来可整合生物标志物数据
- 对比 MILTON（67 个血液标志物模型），Delphi 在大多数疾病上 AUC 更高
- LLama3.1 (8B) 的 AUC 显著低于 Delphi（CVD: 0.63, 痴呆: 0.67, 死亡: 0.69）

### 4.2 生成未来健康轨迹

> **[Figure 3: 生成式建模]**
> Caption: Delphi-2M 从 60 岁开始采样健康轨迹。(b) 模拟 vs 观测的 70-75 岁发病率；(c) 正确预测 token 的比例；(d) 吸烟/饮酒/BMI 亚组的 fold change。

**实验设计**：
- 对验证集 63,662 人，使用 60 岁前数据作为 prompt
- 采样一条未来轨迹，与实际观测对比

**结果**：
- 70-75 岁的人群级发病率被良好复现
- 首年 **17%** 的 token 被正确预测（基线 12-13%），20 年后降至 **14%**
- **吸烟/饮酒/BMI 亚组的差异被准确预测**（fold change 模拟值与观测值一致）
- 从出生开始的完全合成轨迹也能复现性别和年龄特异的发病模式

### 4.3 合成数据训练

**重要发现**：在完全合成数据上训练的 Delphi，验证 AUC 为 **0.74**（原始 0.77），仅差 3 个百分点。

→ 合成数据保留了训练 Delphi 所需的大部分信息，可作为隐私保护的替代数据源。

### 4.4 可解释性分析

> **[Figure 4: 可解释性]**
> Caption: (a) Token embedding UMAP；(b) 个体 SHAP 分析示例；(c) 疾病间 SHAP 效应矩阵；(d) 疾病对死亡率的时间衰减效应。

#### Embedding 空间分析 (UMAP)
- 疾病按 ICD-10 章节自然聚类——模型从数据中学到了疾病分类学
- 显著例外：女性生殖系统癌前病变与乳腺癌聚类；糖尿病与视网膜病变、神经病变聚类
- 高急性死亡率的疾病（心梗、脓毒症）与死亡 token 聚类

#### SHAP 分析
- 100,639 条验证轨迹的系统性 SHAP 分析
- **疾病效应多为同 ICD-10 章内**（共病聚类）
- **精神障碍 (Chapter V)** 和 **妊娠相关 (Chapter XV)** 显示全章范围的聚类效应
- 效应通常是**对称的**（A 影响 B 的程度 ≈ B 影响 A）

#### 时间依赖效应 (Figure 4d)
- **癌症**：对死亡率的影响持续衰减，半衰期数年（反映复发风险和治疗影响）
- **脓毒症**：影响迅速消退，几年内恢复至人群平均水平
- **抑郁**：影响持续
- 这些推断与传统 Nelson-Aalen 分析一致，验证了 Delphi 的时间建模能力

### 4.5 外部验证（丹麦数据）

- **平均 AUC = 0.67** (s.d. 0.09)，略低于 UKB 纵向测试 (0.69)
- 各疾病 AUC **跨数据集 Pearson 相关 0.76** (95% CI: 0.72-0.80)
- **零调参**直接迁移，证明模型学到的共病模式具有跨国泛化性
- 性能下降可归因于：UKB 人群选择偏差、丹麦缺少生活方式数据、编码差异

### 4.6 偏差分析

> **[Figure 5: 偏差分析]**
> Caption: UK Biobank 数据中的流行病学偏差及其在 Delphi-2M 预测中的反映。

**三类偏差**：

1. **健康志愿者偏差 + 入组选择偏差**
   - UKB 入组年龄 40-70 岁 → 入组前死亡不被记录 → immortal time bias
   - 白人英国人过度代表，参与者平均更富裕、教育水平更高
   - Delphi 复现了族裔间的疾病率差异和社会剥夺指数的影响

2. **数据源缺失偏差**
   - 不同数据源（自报、GP、住院、死亡登记）对不同参与者的覆盖不一致
   - 住院记录特有的疾病（如脓毒症 93% 来自住院），在有住院历史的个体中预测率高 8-10 倍
   - 这是**数据收集过程的伪影**，而非真实的疾病关联

3. **年龄和随访限制**
   - 80 岁以上数据有限，该年龄段不可靠建模

## 5. Discussion & 局限性

### 论文自述的局限性

1. **UK Biobank 偏差**：健康志愿者效应、人群不代表性、数据源不一致
2. **数据源伪影**：缺失模式被模型学习为疾病关联，可能产生虚假预测
3. **因果推断局限**：Delphi 建模的是时间依赖关系而非因果关系，不应用于指导干预
4. **年龄范围限制**：80 岁以上数据不足，此年龄段预测不可靠
5. **Token 粒度限制**：仅使用 ICD-10 Level 3 码，丢失了更细粒度的诊断信息

### 论文提出的未来方向

- 整合更多数据模态：处方记录、血液检测、基因组数据、代谢组学、可穿戴设备
- 与 LLM 集成：Delphi 作为 healthcare LLM 的数值推理后端（类似 RAG 减少幻觉）
- 自由文本学习：从非结构化病历中直接学习
- 临床决策支持：筛查入组、风险分层

## 6. 个人思考与组会讨论点

### 值得肯定的设计

1. **优雅的数学框架**：竞争指数模型天然对应 softmax 分类 + 指数等待时间，数学上完美自洽
2. **Padding token 的巧妙设计**：用简单的方法解决了连续时间序列中长间隔的难题
3. **极强的外部验证**：在 190 万丹麦人口上零调参验证，AUC 仅下降 2%
4. **合成数据的惊人表现**：全合成训练仅损失 3% AUC，为隐私保护数据共享提供了实证
5. **2.2M 参数即达最优**：证明健康数据的 scaling law 天花板远低于自然语言，小模型就够了
6. **全面的偏差分析**：主动暴露并分析模型学到的偏差，而不是回避

### 可深入讨论的问题

1. **2M 参数是否太小？**
   - 模型容量受限于 UK Biobank 的 50 万人规模
   - 如果数据量扩大（如全英国 NHS 数据），是否能训练更大、更强的模型？
   - 当前 1,258 token 的词汇量是否过于粗糙？ICD-10 Level 4/5 会如何？

2. **生活方式 token 时间随机化的影响**
   - 将 BMI/吸烟/饮酒的时间点在 ±20~+40 年内随机化是合理的工程选择
   - 但这可能削弱了生活方式变化的真实时间效应（如戒烟后风险下降的时间模式）

3. **数据源偏差是根本性问题**
   - 有住院历史的人被预测更高的疾病率（很多是数据可见性而非真实共病）
   - 这个问题在所有 EHR-based 模型中都存在，Delphi 的贡献是**显式暴露**了它
   - 如何去偏？Delphi 未提供解决方案

4. **与 foundation model 范式的关系**
   - Delphi-2M 只有 2.2M 参数，与 LLM "大力出奇迹" 的范式相反
   - 这说明健康数据的结构远比自然语言简单——token 种类有限、序列较短
   - 但这也意味着 Delphi 的能力上限受制于输入数据的信息量

5. **临床部署的可行性**
   - 预测 >1,000 种疾病的发病率 → 如何呈现给临床医生？信息过载风险
   - 缺乏不确定性量化（置信区间只在 AUC 层面提供）
   - 个体预测的 calibration 如何？（论文只展示了群体水平的 calibration）

6. **与 LPFM/PixCell 等病理 FM 的潜在交集**
   - Delphi 从 EHR 预测疾病轨迹；病理 FM 从图像辅助诊断
   - 两者是否可以互补？如 Delphi 识别高风险个体 → 病理 FM 辅助精确诊断
   - 多模态整合（Delphi embedding + 病理图像特征）是否有价值？
