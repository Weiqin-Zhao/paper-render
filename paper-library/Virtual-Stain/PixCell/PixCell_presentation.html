Motivation & Problem

判别式基础模型的三个结构性瓶颈

Three Structural Bottlenecks of Discriminative Foundation Models

计算病理领域目前以 SSL / VLM 等判别式基础模型为主流（UNI、Virchow、Hoptimus 等），但存在三个它们无法解决的问题：

Computational pathology is currently dominated by discriminative foundation models (UNI, Virchow, Hoptimus, etc.), but three structural problems remain unsolvable:

1. 标注稀缺
1. Annotation Scarcity

Patch 级标注依赖病理学家，成本高、规模受限。小数据集（如 BACH 仅 218 训练样本）严重过拟合。

Patch-level annotation requires pathologists — expensive and unscalable. Small datasets (e.g., BACH with only 218 training samples) suffer severe overfitting.

2. 隐私壁垒
2. Privacy Barriers

机构间数据共享受限于 IRB / 患者隐私 / 数据所有权，各站点只能在自有数据上训练，泛化性差。

Cross-institution data sharing is limited by IRB / patient privacy / data ownership, forcing each site to train on local data with poor generalization.

3. 生成任务超出判别模型能力
3. Generation Beyond Discriminative Models

Virtual staining（H&E → IHC）本质上需要从参考图像合成全新图像，判别式模型无法胜任。

Virtual staining (H&E → IHC) inherently requires synthesizing new images from reference — beyond discriminative capability.

PixCell 的定位

PixCell's Positioning

构建首个病理图像生成式基础模型，用一个统一的生成模型同时应对：数据增强、隐私保护数据共享、Virtual IHC Staining。

Build the first generative foundation model for pathology, unifying: data augmentation, privacy-preserving data sharing, and virtual IHC staining.

核心 Insight：自然图像生成依赖大规模 image-text pairs（LAION-5B 有 58.5 亿对），但病理领域不存在这种数据。PixCell 用 SSL encoder 的 embedding 替代 text caption 作为条件信号，绕开瓶颈，实现无标注大规模条件训练。 Core Insight: Natural image generation relies on massive image-text pairs (LAION-5B: 5.85B pairs), but pathology lacks such data. PixCell uses SSL encoder embeddings instead of text captions as conditioning, bypassing this bottleneck for large-scale annotation-free training.

SD3 VAE 的 16-dim latent 重建质量显著更高（PSNR +4.86 dB）。对于高频纹理丰富的病理图像，VAE 重建精度是生成质量的理论上限。VAE 全程 frozen，训练前一次性预提取所有 latent。 SD3 VAE's 16-dim latent yields significantly better reconstruction (PSNR +4.86 dB). 为什么选 DiT 而非 U-Net？

Why DiT over U-Net?

DiT 优势
DiT Advantages

同构堆叠 scaling（加深 = 更多 block），直接继承 LLM/ViT 的 scaling laws。核心运算（矩阵乘法、attention）对 GPU 高度友好。Cross-attention 天然支持变长条件输入。

Homogeneous stacking (deeper = more blocks), inherits LLM/ViT scaling laws. Core ops (matmul, attention) are GPU-friendly. Cross-attention natively supports variable-length conditions.

U-Net 局限
U-Net Limitations

Scaling 涉及多个耦合轴（分辨率层级数、通道数、attention 位置、skip connection）。异构运算（多分辨率卷积、concat、上下采样）硬件优化更困难。

Scaling involves coupled axes (resolution levels, channels, attention placement, skip connections). Heterogeneous ops (multi-res conv, concat, up/downsampling) harder to optimize on hardware.

3.2 Progressive Training Strategy

不直接在 1024×1024 上训练，而是从低分辨率逐步提升。先学局部细胞/纹理，再学远距离组织架构一致性。

Instead of training directly at 1024×1024, progressively increase resolution. Learn local cell/texture first, then long-range tissue architecture.

3.3 CFG 的特殊性

3.3 CFG Specifics

训练时以 10% 概率随机 drop condition embedding（设为 0），推理时用标准 CFG 公式：

During training, condition embedding is dropped with 10% probability (set to zero). At inference, the standard CFG formula is applied:

ε̂(x_t, c) = ε(x_t, ∅) + w · [ε(x_t, c) − ε(x_t, ∅)]

公式拆解：

Formula Breakdown:

  • ε(x_t, ∅) — 无条件预测：模型在没有任何参考图像条件下，对当前噪声 x_t 预测的去噪方向。可以理解为"模型对病理图像的通用先验"
  • ε(x_t, c) — 有条件预测：模型在给定 UNI-2h embedding c 作为条件下，对 x_t 预测的去噪方向。"模型认为这张参考图像对应的生成方向"
  • ε(x_t, c) − ε(x_t, ∅) — 条件引导信号：两者之差就是"条件带来的额外信息"，即条件相对于无条件的偏移方向
  • w (guidance scale) — 放大系数：控制条件信号的强度。w=1 等于只用有条件预测；w>1 则放大条件信号，让生成结果更忠实于条件；w 过大则 over-amplify 导致伪影
  • ε̂(x_t, c) — 最终去噪方向：从无条件基线出发，沿条件方向走 w 步。本质上是"先看通用先验，再加上 w 倍的条件偏移"
  • ε(x_t, ∅) — Unconditional prediction: the model's denoising direction for noisy x_t with no reference image. Think of it as "the model's general prior for pathology images"
  • ε(x_t, c) — Conditional prediction: the model's denoising direction given UNI-2h embedding c as condition. "Where the model thinks generation should go for this reference"
  • ε(x_t, c) − ε(x_t, ∅) — Guidance signal: the difference is "the extra information from the condition" — the directional offset that the condition adds
  • w (guidance scale) — Amplification factor: controls condition signal strength. w=1 uses only conditional prediction; w>1 amplifies the signal for stronger condition fidelity; too large w causes artifacts from over-amplification
  • ε̂(x_t, c) — Final denoising direction: starting from the unconditional baseline, move w steps along the condition direction. Essentially "general prior + w × conditional offset"

为什么最优 scale 这么低？ Text caption 描述极其多样的图像 → 条件信号模糊 → 需要强 guidance。UNI embedding 只对应高度相似的少量图像 → 条件已经很精确 → 过强 guidance 导致 mode collapse / 伪影。PixCell-1024 的 w 更低（1.2 vs 2.0），因为 16 个空间排列的 embedding 比单个 embedding 提供了更精确的条件。 Why so low? Text captions describe extremely diverse images → vague conditioning → needs strong guidance. UNI embeddings correspond to only highly similar images → already precise → over-guidance causes mode collapse / artifacts. PixCell-1024 uses even lower w (1.2 vs 2.0) because 16 spatially-arranged embeddings are more precise than a single one.

3.4 数据增强策略

3.4 Data Augmentation Strategies

Vanilla Augmentation
Vanilla Augmentation

直接：真实图像 → UNI-2h embedding → PixCell 条件生成 → 合成图像继承原图标签。训练集 = 真实 + 合成 (double size)。当下游 encoder 与 UNI-2h 相同或相似时效果最好。

Direct: real image → UNI-2h embedding → PixCell conditioned generation → synthetic inherits label. Training set = real + synthetic (doubled). Best when downstream encoder matches UNI-2h.

Test-time Constraint-Guided
Test-time Constraint-Guided

当下游 encoder（如 Virchow-2）与 UNI-2h 不同时，在去噪过程中施加额外梯度约束：推动生成结果在目标 encoder 空间与参考图像更相似。通过 grid search 选择最优 (策略, guidance, lr, wd)。

When downstream encoder (e.g., Virchow-2) differs from UNI-2h, add gradient constraint during denoising: push generated results to be more similar in target encoder space. Optimal (strategy, guidance, lr, wd) selected via grid search.

Rectified Flow：Embedding Space Domain Transfer

Rectified Flow: Embedding Space Domain Transfer

学习从 H&E embedding 分布到 IHC embedding 分布的映射。使用 Rectified Flow 建模分布到分布的映射（而非一对一回归），保留多样性。

Learns mapping from H&E embedding distribution to IHC embedding distribution. Uses Rectified Flow to model distribution-to-distribution mapping (not one-to-one regression), preserving diversity.

  • 网络结构：残差 MLP（简单高效）
  • 每种 IHC stain 单独训练一个 Rectified Flow（HER2、ER、PR、Ki67）
  • 训练：500 epochs, batch 512, lr=1e-4
  • 推理：Euler ODE solver 积分 100 步
  • Architecture: Residual MLP (simple and efficient)
  • One Rectified Flow per IHC stain (HER2, ER, PR, Ki67)
  • Training: 500 epochs, batch 512, lr=1e-4
  • Inference: Euler ODE solver, 100 integration steps
关键优势：整个 pipeline 在 patch-level embedding 空间操作，不要求 pixel-perfect alignment。对 MIST 这类粗略配对数据（相邻切片，天然存在位移）非常友好。 Key advantage: The entire pipeline operates in patch-level embedding space, requiring no pixel-perfect alignment. Very friendly to loosely-paired data like MIST (adjacent sections with natural displacement).

  • Only add low-rank adapters to DiT's cross-attention layers
  • One LoRA per IHC stain

数据增强的增益在小数据集尤为显著。具体流程如下：

Augmentation gains are especially significant on small datasets. The specific pipeline:

"+增强" 列的具体做法：

How the "+ Augment" column works:

  1. 提取 embedding：对训练集中每张真实图像，用 UNI-2h 提取其 embedding
  2. 条件生成：将该 embedding 作为条件，送入 PixCell 从随机噪声生成一张语义一致但像素不同的合成图像
  3. 继承标签：合成图像自动继承原图的类别标签（因为 embedding 保证了语义一致性）
  4. 合并训练：训练集 = 原始真实图像 + 合成图像（数据量翻倍），在此基础上训练 linear probe classifier
  1. Extract embedding: For each real image in the training set, extract its UNI-2h embedding
  2. Conditional generation: Feed this embedding as condition into PixCell, generating a semantically consistent but pixel-different synthetic image from random noise
  3. Inherit label: The synthetic image automatically inherits the original's class label (embedding guarantees semantic consistency)
  4. Merge & train: Training set = original real images + synthetic images (doubled), then train a linear probe classifier on top
论文还提供了 Test-time Constraint-Guided 变体（见 3.4 节）：当下游 encoder 与 UNI-2h 不同时，在生成过程中额外施加目标 encoder 空间的梯度约束，确保合成图像在目标 encoder 看来也与原图相似。最终通过 grid search 为每个 (encoder, dataset) 组合选择最优策略。 The paper also provides a Test-time Constraint-Guided variant (see Section 3.4): when the downstream encoder differs from UNI-2h, an additional gradient constraint in the target encoder's space is applied during generation, ensuring synthetic images are also similar to originals as seen by the target encoder. The optimal strategy is selected per (encoder, dataset) pair via grid search.

可深入讨论的问题

Topics for Discussion

1. 合成数据的"天花板效应"1. "Ceiling Effect" of Synthetic Data

同分布合成数据翻倍没有增益（76.76% vs 77.89%），说明不能提供新信息。只有引入新分布（HistAI）才有提升。合成数据的价值在于跨机构分布的模拟，而非简单的数据量翻倍。这对我们使用合成数据增强的场景有什么启示？

Same-distribution doubling shows no gain (76.76% vs 77.89%), indicating no new information. Only new-distribution (HistAI) helps. Synthetic data's value lies in simulating cross-institutional distributions, not simple dataset doubling. What implications for our augmentation strategy?

2. 隐私保证的实际强度2. Actual Strength of Privacy Guarantees

论文承认生成模型可能 leak 训练数据（Carlini et al. 2023）。合成数据共享仅是"第一步"，不能替代严格的隐私保护方法（如 differential privacy）。条件生成可能比无条件生成泄露更多信息——embedding 本身是否包含可识别的患者信息？

Paper acknowledges models may leak training data (Carlini et al. 2023). Synthetic sharing is just "a first step", not a replacement for strict privacy methods (e.g., differential privacy). Conditioned generation may leak more than unconditional — do embeddings contain identifiable patient information?

3. Virtual Staining 的 Pipeline 依赖链3. Virtual Staining Pipeline Dependencies

UNI-2h 编码能力 → Rectified Flow 质量 → LoRA 适配 → 最终 IHC 质量。任一环节出问题都会影响最终结果。如果目标 stain 是 UNI-2h 训练时未见过的稀有 stain，整个 pipeline 可能失效。

UNI-2h encoding → Rectified Flow quality → LoRA adaptation → final IHC quality. Any weak link affects the output. If the target stain is rare/unseen by UNI-2h, the entire pipeline may fail.

4. 诊断评估的可靠性4. Reliability of Diagnostic Evaluation

DeepLIIF 与病理学家在 PR (κ=0.516) 和 Ki67 (κ=0.554) 上一致性偏低。论文的诊断评估结果受限于 DeepLIIF 的准确性。更严格的验证应完全基于病理学家评估。

DeepLIIF-pathologist agreement is low for PR (κ=0.516) and Ki67 (κ=0.554). Diagnostic evaluation is bounded by DeepLIIF accuracy. Stricter validation should be fully pathologist-based.

5. PanCan-Test Split 策略5. PanCan-Test Split Strategy

Patch-level split 可能导致 data leakage（同一 WSI 的相邻 patches 极其相似）。好在论文还在 SPIDER 和 Non-H&E 上做了 OOD 评估，部分缓解了担忧。

Patch-level split may cause data leakage (adjacent patches from same WSI are very similar). Mitigated by OOD evaluation on SPIDER and Non-H&E datasets.

6. 与 LPFM 的对比与互补6. Comparison and Complementarity with LPFM

LPFM 定位为低层级病理基础模型（修复+虚拟染色），PixCell 为生成式基础模型。Virtual staining 是两者的交集：LPFM 在像素空间做条件生成（需 paired data），PixCell 在 embedding 空间做 domain transfer（只需 loosely-paired data）。两者是否可以结合？如 PixCell 做初步 stain transfer → LPFM 做质量精炼。

LPFM: low-level pathology foundation model (restoration + virtual staining); PixCell: generative foundation model. Virtual staining is their intersection. LPFM works in pixel space (needs paired data); PixCell in embedding space (loosely-paired sufficient). Could they combine? e.g., PixCell for initial transfer → LPFM for refinement.
